1. HTML Basics

What is HTML?

HTML (HyperText Markup Language) is the standard markup language for creating web pages.

Basic HTML Document Structure

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Document Title</title> </head> <body> <!-- Content goes here --> </body> </html>
Explanation
  • <!DOCTYPE html> - Declares this as an HTML5 document
  • <html> - Root element of the HTML page
  • <head> - Contains meta information about the document
  • <meta charset="UTF-8"> - Defines character encoding
  • <title> - Specifies the title shown in browser tab
  • <body> - Contains the visible page content

Note: The <!DOCTYPE html> declaration must be the very first thing in your HTML document.

Warning: Some elements are empty (void) elements and don't have closing tags, like <img> and <br>.

Exercise 1: Create Your First HTML Page

Create a basic HTML page with a heading and paragraph.

<!DOCTYPE html> <html> <head> <title>My First Page</title> </head> <body> <h1>Welcome to My Website</h1> <p>This is my first HTML page.</p> </body> </html>

4. Links & Anchors

The <a> Tag

The <a> tag defines a hyperlink, which is used to link from one page to another.

Basic Link Syntax

<a href="url">link text</a>

Link Types

<!-- Absolute URL --> <a href="https://www.example.com">Visit Example</a> <!-- Relative URL --> <a href="about.html">About Us</a> <!-- Email link --> <a href="mailto:someone@example.com">Send Email</a> <!-- Phone link --> <a href="tel:+1234567890">Call Us</a> <!-- Link to page section --> <a href="#section2">Go to Section 2</a>

Visit Example

About Us (links back to this section)

Send Email

Call Us

Link Attributes

Attribute Description
href Specifies the URL of the page the link goes to
target Specifies where to open the linked document (_blank, _self, _parent, _top)
download Specifies that the target will be downloaded when clicked
rel Specifies the relationship between the current and linked document
title Specifies extra information about the link (shown as tooltip)

Link with Attributes Example

<a href="https://www.example.com" target="_blank" rel="noopener noreferrer" title="Visit Example Website"> Open Example in New Tab </a>

Open Example in New Tab

Security Best Practice: When using target="_blank", always include rel="noopener noreferrer" to prevent potential security vulnerabilities.

6. Images & Media

The <img> Tag

The <img> tag is used to embed images in HTML pages.

Basic Image Syntax

<img src="image.jpg" alt="Description of image">

Image Attributes

Attribute Description

9. Semantic HTML

Semantic HTML introduces meaning to the web page rather than just presentation.

Semantic Page Layout Example

<header> <h1>Website Title</h1> <nav> <a href="/">Home</a> <a href="/about">About</a> </nav> </header> <main> <article> <h2>Article Title</h2> <p>Article content...</p> </article> <aside> <h3>Related Links</h3> <ul> <li><a href="#">Link 1</a></li> </ul> </aside> </main> <footer> <p>Copyright 2023</p> </footer>

Semantic Elements

Element Purpose
<header> Introductory content or navigation
<nav> Navigation links
<main> Main content of the document
<article> Self-contained composition
<section> Thematic grouping of content
<aside> Content indirectly related to main content
<footer> Footer for its nearest section
<figure> Self-contained content with caption
<figcaption> Caption for a figure
<time> Machine-readable date/time
<mark> Highlighted text
<details> Additional details that can be shown/hidden
<summary> Visible heading for a details element

Benefits of Semantic HTML:

  • Better accessibility
  • Improved SEO
  • Easier to maintain
  • More meaningful structure

10. Metadata

The <head> Element

The <head> element contains meta information about the HTML document.

<head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <meta name="description" content="Free Web tutorials"> <meta name="keywords" content="HTML, CSS, JavaScript"> <meta name="author" content="John Doe"> <title>Document Title</title> <link rel="stylesheet" href="styles.css"> <link rel="icon" href="favicon.ico" type="image/x-icon"> </head>

Common Meta Tags

Tag Purpose
<meta charset="UTF-8"> Defines character encoding
<meta name="viewport" content="width=device-width, initial-scale=1.0"> Controls viewport for responsive design
<meta name="description" content="..."> Provides a description for search engines
<meta name="keywords" content="..."> Provides keywords for search engines
<meta name="author" content="..."> Specifies the author of the document
<meta http-equiv="refresh" content="30"> Refreshes the page every 30 seconds

The <link> Tag

The <link> tag defines the relationship between the current document and an external resource.

<!-- Link to CSS file --> <link rel="stylesheet" href="styles.css"> <!-- Favicon --> <link rel="icon" href="favicon.ico" type="image/x-icon"> <!-- Preload important resources --> <link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>

The <style> Tag

The <style> tag is used to define style information (CSS) for a document.

<style> body { background-color: lightblue; } h1 { color: navy; margin-left: 20px; } </style>

The <script> Tag

The <script> tag is used to embed or reference JavaScript code.

<!-- Internal JavaScript --> <script> function myFunction() { document.getElementById("demo").innerHTML = "Hello JavaScript!"; } </script> <!-- External JavaScript --> <script src="myscript.js"></script>

The <div> tag defines a division or section in an HTML document, used as a container for other elements.

12. Embedded Content

The <iframe> Tag

The <iframe> tag specifies an inline frame for embedding external content.

<iframe src="https://www.example.com" width="600" height="400" title="Example iframe" frameborder="0" ></iframe>

The <embed> Tag

The <embed> tag defines a container for an external resource.

<embed type="application/pdf" src="document.pdf" width="600" height="400" >

The <object> Tag

The <object> tag defines an embedded object within an HTML document.

<object data="movie.swf" width="400" height="300" ></object>

The <picture> Tag

The <picture> tag gives more flexibility in specifying image resources.

<picture> <source media="(min-width: 650px)" srcset="img_pink_flowers.jpg"> <source media="(min-width: 465px)" srcset="img_white_flower.jpg"> <img src="img_orange_flowers.jpg" alt="Flowers"> </picture>

14. HTML Entities and Special Characters

HTML entities are used to display reserved characters or characters that don't appear on your keyboard.

Why Use Entities?

  • Display characters that have special meaning in HTML (< > &)
  • Show characters not available on your keyboard
  • Display invisible characters (non-breaking space)
  • Show characters that might not display correctly with the document's character encoding

Common HTML Entities

Character Entity Name Entity Number Description
< &lt; &#60; Less than
> &gt; &#62; Greater than
& &amp; &#38; Ampersand
" &quot; &#34; Double quotation mark
' &apos; &#39; Single quotation mark
  &nbsp; &#160; Non-breaking space
© &copy; &#169; Copyright symbol
® &reg; &#174; Registered trademark
€ &euro; &#8364; Euro currency
♥ &hearts; &#9829; Heart symbol

Entity Usage Examples

<p>To display a &lt;div&gt; tag, you write &amp;lt;div&amp;gt;</p> <p>Copyright &copy; 2023 My Company</p> <p>Price: 15&euro; (fifteen euros)</p>

To display a <div> tag, you write &lt;div&gt;

Copyright © 2023 My Company

Price: 15€ (fifteen euros)

Tip: For a complete list of HTML entities, refer to the W3C HTML5 Character Reference.

15. HTML Comments

Comments are used to include notes in the code that are not displayed in the browser.

Comment Syntax

<!-- This is a comment -->

Uses of Comments

  • Explaining code functionality
  • Temporarily disabling code (commenting out)
  • Leaving notes for other developers
  • Creating documentation within code

Comment Examples

<!-- Main navigation starts here --> <nav> <ul> <li><a href="/">Home</a></li> <!-- Temporarily removing about link <li><a href="/about">About</a></li> --> <li><a href="/contact">Contact</a></li> </ul> </nav> <!-- Main navigation ends here -->
  • Home
  • Contact

Warning: Comments are visible in the page source, so don't include sensitive information in them.

Conditional Comments (Deprecated)

Previously used for Internet Explorer-specific code (no longer supported in IE10+):

<!--[if IE]> <p>This content is only visible in Internet Explorer</p> <![endif]-->

Modern Alternative: Use feature detection with JavaScript or CSS @supports instead of conditional comments.

17. HTML Validation and Best Practices

HTML Validation

Validating your HTML ensures it follows the official standards and helps avoid problems.

Ways to Validate HTML:

  • W3C Markup Validation Service
  • Browser developer tools
  • IDE/editor plugins
  • Build tools like HTMLHint

HTML Best Practices

Do's and Don'ts

Best Practice Why It's Important
Use proper document structure Ensures browser compatibility and accessibility
Always include alt attributes for images Accessibility and SEO benefits
Use semantic HTML5 elements Better structure and meaning for content
Separate content (HTML), presentation (CSS), and behavior (JS) Maintainable, scalable code
Validate your HTML regularly Catches errors early
Use lowercase for tags and attributes Consistency and XHTML compatibility
Quote all attribute values Prevents parsing errors
Close all tags properly Avoids rendering issues
Don't use deprecated elements Future compatibility
Don't use tables for layout Use CSS for layout instead

Accessibility Considerations

  • Use proper heading hierarchy (<h1> to <h6>)
  • Provide text alternatives for non-text content
  • Ensure sufficient color contrast
  • Use ARIA attributes when needed
  • Make interactive elements keyboard-navigable
